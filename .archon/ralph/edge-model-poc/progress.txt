## Codebase Patterns

### Detector predict() contract
- Where: `server/vision/detector.py:7-40` (`CatDetector.detect`)
- Pattern: every model wrapper exposes
  `predict(frame: np.ndarray) -> list[{"bbox": [x1,y1,x2,y2 normalized], "confidence": float}]`.
  The edge eval harness adapters mirror this exact shape so format swaps
  are uniform.

### pytest_asyncio collection bug in this repo
- Where: pytest-asyncio 0.23 + pytest 8 + a tests `__init__.py` package.
- Pattern: `pytest server/tests/` and `pytest training/edge/tests/` both
  fail at COLLECTION with `AttributeError: 'Package' object has no attribute 'obj'`.
  The fix is to disable the asyncio plugin: `pytest -p no:asyncio …`.
  Pre-existing; not specific to anything in this PRD.

### ultralytics fuses Conv+BN at first predict() — measure params/FLOPs first
- Where: `training/edge/eval/adapters/pytorch_adapter.py:_compute_params_flops`
- Pattern: `YOLO(...)` followed by `model(frame)` mutates `model.model` in
  place (Conv+BN fusion + reparameterization). thop's hook tracing breaks
  on the fused graph and param counts shift. Snapshot params and FLOPs in
  `__init__` BEFORE any predict() runs.

### Val sets are big — manifest > duplication
- Where: `training/edge/data/val/manifest.txt` + `training/edge/eval/val_loader.py`
- Pattern: The naive bootstrap that copies images alongside labels
  ballooned the repo by 68 MB. Better: write a manifest of
  tab-separated `image_path<TAB>label_path` (paths relative to manifest's
  dir) and resolve at load time. Keeps repo tracking only the labels
  (~100 KB) while images stay where they live (`data/cat_photos/`).

### Run eval harness end-to-end
- Where: `training/edge/eval/README.md`
- Pattern: any later story producing a model variant should write its
  result through `python -m training.edge.eval.run_eval --story-id <id> ...`
  to `training/edge/results/<id>.json` matching the EvalResult schema.
  Future SUMMARY.md (US-012) is built mechanically from these files.

**Learnings**:
- Silver GT mAP for yolov8s is 1.0 by construction (the val labels are its
  own predictions). Document this so it doesn't get cited as a real number.
- Saved val-set images would have committed 68 MB of duplicated JPEGs;
  a manifest pointing back at `data/cat_photos/` is the right design.
- ultralytics 8.2.0 mutates the model in place during the first predict()
  via Conv+BN fusion. Compute params/FLOPs in __init__ before any
  predict() to keep thop happy and the param count consistent.
- pytest_asyncio's collection bug bites BOTH server/tests/ and
  training/edge/tests/ in this repo. Recommended invocation is
  `pytest -p no:asyncio …`. We did not "fix" the broken plugin in this
  story since it's out of scope.
- run_eval flops=0 is the silent-fallback signal that thop is missing or
  the adapter couldn't trace; pytorch path fills it correctly.
