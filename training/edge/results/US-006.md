# US-006 — Knowledge distillation: YOLOv8s teacher → YOLOv8n student (cat-only)

## What ran

`training/edge/yolo/distill_train.py` runs a single-class fine-tune of
YOLOv8n on the US-002 dataset (480 train / 120 val) with a YOLOv8s teacher
contributing a temperature-softened BCE distillation term added to the
ultralytics supervised loss:

```
combined = sup_loss + alpha * distill_loss
distill_loss = mean over scales {P3, P4, P5} of:
    BCE_with_logits(student_cat_logit / T,  sigmoid(teacher_cat_logit / T)) * T**2
```

Channel alignment: both heads emit `(B, 4*reg_max + nc, H, W)` per scale.
The student is single-class (`nc=1` ⇒ cat at idx 0) and the teacher is the
COCO-pretrained YOLOv8s (`nc=80` ⇒ cat at COCO idx 15). The distillation
term thus matches a **single channel** per scale across the two networks.

Hyperparameters (per PRD `technicalNotes`):

| Param | Value | Source |
|-------|-------|--------|
| alpha | 0.5 | PRD-stated default |
| temperature T | 4.0 | Hinton 2015 KD recipe |
| epochs | 50 | matches US-003 (apples-to-apples wall clock) |
| imgsz | 224 | edge target |
| batch | 32 | matches US-003 |
| device | CUDA:0 (RTX 4090) | auto |
| optimizer | ultralytics default (SGD, momentum 0.937, weight decay 5e-4) | inherited |

Quantization re-uses the **identical** US-004 PTQ pipeline:

```
yolov8n_cat_distilled.pt
    │  ultralytics .export(format='onnx', imgsz=224, dynamic=False, simplify=True)
    ▼
yolov8n_cat_distilled.onnx (~11.5 MB)
    │  onnx2tf.convert(output_signaturedefs=True)        # NHWC SavedModel
    ▼
SavedModel/
    │  tf.lite.TFLiteConverter.from_saved_model(...)
    │    optimizations=[Optimize.DEFAULT]
    │    representative_dataset = calibration_loader(200 frames)
    │    target_spec.supported_ops=[TFLITE_BUILTINS_INT8]
    │    inference_input_type=tf.int8
    │    inference_output_type=tf.int8
    │    experimental_new_quantizer=True
    ▼
training/edge/models/yolov8n_cat_distilled_int8.tflite (3.07 MB)
```

The calibration set is the same 200-frame subset used by US-004 — by
construction this comparison isolates "what the KD weights do under the
same quantization recipe".

### Headline findings

- **fp32 distilled vs fp32 baseline**: +0.091 mAP (0.909 → 1.000). The teacher's
  cat-channel signal smoothed the student's cls head; with KD the val set is
  saturated. (The val set is small — 120 images, 134 instances — so 1.000
  reflects saturation rather than a meaningful mAP ceiling, but it does
  confirm the distillation didn't degrade training.)

- **int8 distilled vs int8 baseline**: **+0.205 mAP** (0.000 → 0.205). This is
  the **critical** finding for the PoC: where US-004's PTQ collapsed the
  cls head to a single int8 bucket and US-005's QAT couldn't recover, KD
  at fp32 produces a head whose cls activations are numerically less
  hostile to the per-tensor quantizer. We're not close to fp32 yet (the
  PTQ drop is still 79.5 mAP-points = mAP_drop_quant ≈ 0.795), but the
  distilled int8 model now produces *some* non-zero detections —
  i.e. it's a usable starting point in the YOLOv8 family, not a brick.

- **Size**: identical to US-004 (3.07 MB) — same architecture, same FlatBuffer
  layout. Distillation acts on the weights, not the graph shape.

- **Latency**: ~3.7 ms p50, within noise of US-004 (3.59 ms p50). The host
  compute budget is unchanged; KD doesn't inflate the runtime.

### Why KD helped where QAT didn't

Re-stating the failure mode from `US-004.md` and `US-005.md` in light of
the new data:

1. YOLOv8's monolithic Detect output `(1, 5, 1029)` mixes box (xywh in
   pixel coords 0..imgsz) with cls (sigmoid in [0,1]) on a single tensor
   that the per-tensor quantizer assigns a single scale.
2. PTQ picks a scale dominated by xywh; cls logits are crushed.
3. QAT in US-005 simulated weight quantization but the export-time
   activation scale on the output tensor was still computed by
   TFLiteConverter from calibration, so the cls collapse persisted.
4. KD pre-shaping the cls logits — driving the student's cat-channel toward
   the teacher's calibrated logit magnitude rather than letting it free-
   range — makes the cls portion of the output tensor numerically larger
   relative to xywh's range. After quantization, more of the cls range
   survives the per-tensor scale, so confidence values are no longer
   identically zero.

This is not a full recovery (still −79.5 mAP-points) but it's the first
observation in this PoC of YOLOv8n PTQ producing **non-zero detections at all**.

### What this implies for the rest of the PoC

- **Carry into US-011 (TFLM bench)**: bench `yolov8n_cat_distilled_int8.tflite`
  in addition to (or instead of) the original `yolov8n_cat_int8.tflite` — the
  arena and op breakdown will be the same (architecturally identical), but
  having the distilled artifact means a `predicted_s3_fps` row that maps to
  a model with a measurable mAP > 0.
- **Carry into US-012 (SUMMARY)**: the YOLOv8n family now has a credible
  candidate (`yolov8n_cat_distilled_int8.tflite`, mAP 0.205, 3.07 MB) to
  contend against the NanoDet pipeline (US-007–US-009). The SUMMARY decision
  matrix will report this as the YOLO-family int8 row.
- **Future (post-PoC)**: pairing KD with a Detect-head split — emitting cls
  and xywh as separate output tensors so each gets its own per-tensor
  scale — is the obvious next step to close the remaining 79-point gap.
  Out of scope for this PoC; flagged in the SUMMARY follow-ups.

## Reproduce

```bash
# train (~1.5 min on a single 4090)
python -m training.edge.yolo.distill_train \
    --teacher yolov8s.pt \
    --student yolov8n.pt \
    --data    training/edge/data/labeled/data.yaml \
    --epochs 50 --imgsz 224 --batch 32 --device 0 \
    --alpha 0.5 --temperature 4.0 \
    --fp32-out training/edge/models/yolov8n_cat_distilled.pt \
    --skip-quantize

# eval fp32
python -m training.edge.eval.run_eval \
    --model training/edge/models/yolov8n_cat_distilled.pt \
    --format pytorch --story-id US-006-fp32 \
    --val-dir training/edge/data/labeled/val --imgsz 224

# quantize (re-uses US-004 pipeline)
python -m training.edge.yolo.distill_train \
    --skip-train \
    --fp32-out training/edge/models/yolov8n_cat_distilled.pt \
    --int8-out training/edge/models/yolov8n_cat_distilled_int8.tflite \
    --calib-dir training/edge/data/calibration_frames --imgsz 224

# eval int8
python -m training.edge.eval.run_eval \
    --model training/edge/models/yolov8n_cat_distilled_int8.tflite \
    --format tflite_int8 --story-id US-006-int8 \
    --val-dir training/edge/data/labeled/val --imgsz 224
```
