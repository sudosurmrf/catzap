# US-001 — Edge eval harness + YOLOv8s/YOLOv8n baselines

## What landed

- `training/edge/eval/` harness: `types.py` (EvalResult), `metrics.py` (single-class
  mAP@0.5), `latency.py` (p50/p95 over 100 single-thread runs after 10 warmup),
  `val_loader.py`, `bootstrap_val.py`, `run_eval.py` CLI, plus
  `adapters/{pytorch,onnx,tflite}_adapter.py`.
- `training/edge/data/val/` silver-GT val set: 22 frames bootstrapped from
  `data/cat_photos/` via the existing `server.vision.detector.CatDetector`
  (yolov8s, conf >= 0.35). 5 source frames had zero teacher detections and
  were skipped. See `training/edge/eval/README.md` for the procedure.
- `training/edge/requirements_edge.txt` pins `onnx`, `onnxruntime`,
  `tensorflow-cpu>=2.15`, `pycocotools`, `thop`, `onnx-tf`.
- 8 pytest cases under `training/edge/tests/test_eval_harness.py`.

## Baseline rows

Measured with single-thread CPU on the dev workstation; latency is p50
over 100 timed inferences (10 warmup) at the listed input size.

| Story ID                          | Model       | imgsz | mAP@0.5 | Size (MB) | Params (M) | FLOPs (G) | p50 (ms) | p95 (ms) |
|-----------------------------------|-------------|-------|---------|-----------|------------|-----------|----------|----------|
| US-001-yolov8s-baseline           | yolov8s.pt  | 1280  | 1.000   | 21.53     | 11.17      | 57.63     | 9.57     | 11.15    |
| US-001-yolov8n-baseline-1280      | yolov8n.pt  | 1280  | 0.866   | 6.23      | 3.16       | 17.72     | 8.99     | 10.49    |

## Caveats

- **Silver GT, not gold.** The val labels are predictions made by the same
  yolov8s model that we use as the s-baseline. Its mAP is therefore 1.0 by
  construction — that does **not** mean yolov8s is perfect. US-002's manual
  cleanup converts this into reviewed labels; rerun US-001 after that to get
  a meaningful s-vs-n delta.
- **Tiny val set (22 images).** mAP numbers carry high variance. Treat them
  as smoke checks for the harness; real comparisons start at US-002.
- **FLOPs via `thop`.** Cached at adapter init, before ultralytics fuses
  Conv+BN at the first predict() call (which throws off thop hooks).
- **Latency at imgsz=1280.** With single-thread CPU + no batching, both nano
  and small are <12 ms p95 because the val frames already start near 224x224
  scale before resizing — these aren't representative of frames captured at
  full 1280 resolution. Real edge latency only matters at the eventual
  on-device imgsz (target 224); the 1280 row is here for protocol parity
  with the existing server detector default.

## Reproduce

```bash
# bootstrap silver val set (one-shot; rerun if you change cat_photos)
python -m training.edge.eval.bootstrap_val \
    --src data/cat_photos --out training/edge/data/val --teacher yolov8s.pt

# baselines
for SID in US-001-yolov8s-baseline US-001-yolov8n-baseline-1280 US-001-yolov8n-baseline-224; do
    case "$SID" in
        *yolov8s*) MODEL=yolov8s.pt; IMGSZ=1280;;
        *1280)     MODEL=yolov8n.pt; IMGSZ=1280;;
        *224)      MODEL=yolov8n.pt; IMGSZ=224;;
    esac
    python -m training.edge.eval.run_eval \
        --model "$MODEL" --format pytorch \
        --story-id "$SID" --val-dir training/edge/data/val --imgsz "$IMGSZ"
done
```
