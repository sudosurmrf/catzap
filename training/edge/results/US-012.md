# US-012 — Final SUMMARY.md decision matrix + recommendation

## What this story produced

- `training/edge/results/SUMMARY.md` — the PRD's decision matrix + a
  recommendation paragraph addressing the five PRD-mandated points
  (winner, predicted S3 fps, PSRAM headroom, sub-5-fps caveat, fp32-vs-INT8
  follow-up trigger).
- `training/edge/results/aggregate_summary.py` — the script that walks the
  per-story result JSONs and renders SUMMARY.md. SUMMARY.md is fully
  regenerable from the raw JSONs; nothing in it is hand-edited.
- `training/edge/tests/test_summary_aggregate.py` — fixture-driven tests
  covering row order, column presence, fp32-row TFLM-column handling,
  optional-row skipping (US-005 absent), required-row missing detection,
  TFLM-incompatible row preservation, recommendation content, and CLI
  round-trip.

## How to read SUMMARY.md

The decision matrix is six rows in the order specified by US-012's
acceptance criterion:

1. `YOLOv8n-cat fp32` — fine-tuned PyTorch checkpoint, no quantization
   (US-003)
2. `YOLOv8n-cat INT8` — PTQ INT8 .tflite (US-004)
3. `YOLOv8n-cat-QAT INT8` — QAT INT8 .tflite (US-005, only present if
   the PTQ mAP drop > 3 points triggered the QAT branch)
4. `YOLOv8n-cat-distilled INT8` — KD student INT8 .tflite (US-006)
5. `NanoDet-cat fp32` — NanoDet-Plus-m fine-tuned to single-class cat
   (US-008; deviation noted in US-008.md — gated on isolated venv)
6. `NanoDet-cat INT8` — NanoDet INT8 .tflite (US-009)

Columns are the seven the PRD spelled out: `size_bytes`, `mAP@0.5`,
`host_int8_latency_ms_p50`, `predicted_s3_fps`, `tflm_arena_bytes`,
`tflm_compatible`. fp32 rows show `—` for the TFLM columns and `n/a` for
`tflm_compatible` so the reader can't confuse "not benchmarked" with "not
compatible". Sizes and arenas are shown as raw bytes plus a parenthesized
MB rendering for readability.

The recommendation paragraph:

- (a) names the model to flash first,
- (b) gives the predicted ESP32-S3 fps (under the 8x x86->S3 multiplier
  documented in `firmware/edge-bench/README.md`),
- (c) reports PSRAM headroom against the ~8 MB target,
- (d) flags whether predicted fps is below the PRD's 5-fps success
  target, with the cat-detection-specific judgment call,
- (e) triggers a follow-up message when the fp32 counterpart beats the
  INT8 winner by >5 points (the "investigate QAT or wider model" hint).

## How to regenerate

```bash
python training/edge/results/aggregate_summary.py \
    --results-dir training/edge/results \
    --output training/edge/results/SUMMARY.md
```

The script is idempotent. It reads the per-story JSONs as-is — refining a
single story (e.g. re-running US-006-int8 with a better calibration set)
and re-running this command will refresh SUMMARY.md.

The TFLM-shape rows pass through `firmware/edge-bench/aggregate.py` rather
than re-parsing the bench-JSON shape locally, so any schema drift in
`run_bench.report_to_result` only needs to be updated in one place. See
the "US-011 aggregator is the US-012 ingestion seam" pattern in
`progress.txt`.

## Honest framing on the current numbers

The PoC's current matrix (committed under `training/edge/results/`) shows:

- YOLOv8n-cat fp32 mAP 0.91 collapses to 0.00 under PTQ (US-004) and
  stays at 0.00 under QAT (US-005). The KD student (US-006) is the only
  YOLO INT8 variant that retains non-trivial mAP (0.205) — that's the
  recommended flash candidate.
- NanoDet INT8 mAP is 0.00 by design of the current export pipeline (per
  US-009.md the per-tensor INT8 output scale collapses cls scores; this
  is a documented PoC finding, not a bug in the eval harness).
- All four INT8 .tflite candidates load cleanly under TFLM (no
  unsupported ops). PSRAM is not the constraint — the worst-case arena
  is NanoDet at 2.15 MB, leaving ~5.85 MB headroom in the 8 MB budget.
- Predicted ESP32-S3 fps is well below 5 for every candidate (0.07-0.11
  fps under the 8x multiplier). The recommendation paragraph treats this
  as expected for a static-cat monitoring rig and flags re-measurement
  on real silicon as a follow-up.

The recommendation deliberately avoids overclaiming. The follow-up text
hard-codes the rule: "if fp32 beats INT8 by >5 points, recommend
investigating QAT or wider student" — the script computes the actual
delta and renders the trigger only when warranted.
