# US-005 — Conditional QAT recovery

## Decision

`training/edge/yolo/qat_finetune.py` reads US-003.json and US-004.json:

```
mAP_drop = US-003.map50 - US-004.map50
         = 0.909 - 0.000
         = 0.909  (90.9 mAP points)
```

`90.9 > 3.0` → **QAT branch executed.**

## What ran

One epoch of QAT-flavored fine-tune at `lr0=1e-4`, `lrf=1.0` (flat schedule
across the single epoch), `batch=32`, `imgsz=224`, `single_cls=True`,
device=GPU 0, starting from `training/edge/models/yolov8n_cat.pt`.

Fake-quant insertion via `torch.ao.quantization`:

- ✅ `prepare_qat(model.model, inplace=True)` succeeded with the default
  `fbgemm` qconfig (per-channel weight, per-tensor activation int8).
  Required flipping `model.model.train()` first; ultralytics loads
  checkpoints in eval mode and `prepare_qat` asserts training mode.
- After insertion, ultralytics' `train()` carried the fake-quant graph
  through one epoch of optimizer steps. The QAT-fine-tuned checkpoint
  was copied to `training/edge/models/yolov8n_cat_qat.pt`.

Re-export through the **identical** PTQ pipeline from US-004
(`training.edge.yolo.export_quantize.export_and_quantize`):

```
yolov8n_cat_qat.pt
    │  ultralytics .export(format='onnx', imgsz=224, dynamic=False, simplify=True)
    ▼
yolov8n_cat_qat.onnx (11.5 MB)
    │  onnx2tf.convert(output_signaturedefs=True)   # NHWC SavedModel
    ▼
SavedModel/
    │  tf.lite.TFLiteConverter.from_saved_model(...)
    │    optimizations=[Optimize.DEFAULT]
    │    representative_dataset = calibration_loader(200 frames)
    │    target_spec.supported_ops=[TFLITE_BUILTINS_INT8]
    │    inference_input_type=tf.int8
    │    inference_output_type=tf.int8
    │    experimental_new_quantizer=True
    ▼
training/edge/models/yolov8n_cat_qat_int8.tflite (3.07 MB)
```

## Comparison: PTQ (US-004) vs QAT (US-005)

| Story  | Format                  | mAP@0.5 | size    | latency p50 | latency p95 |
|--------|-------------------------|---------|---------|-------------|-------------|
| US-003 | pytorch fp32 (baseline) | 0.909   | 6.20 MB | 5.91 ms     | 7.22 ms     |
| US-004 | tflite int8 (PTQ)       | 0.000   | 3.07 MB | 3.59 ms     | 3.97 ms     |
| US-005 | tflite int8 (QAT)       | 0.000   | 3.07 MB | 3.81 ms     | 4.45 ms     |

**QAT mAP recovery: 0.000 → 0.000  (Δ +0.000)**

QAT did **not** recover accuracy. This is a real finding, not a script bug.

## Why QAT didn't help

US-004 documented the failure mode in detail: YOLOv8's Detect head emits a
**single** output tensor `(1, 5, 1029)` mixing xywh in input-pixel space
(0..480) with cls in [0,1]. TFLite's per-tensor activation quantizer picks
one scale fitted to xywh; cls collapses to a single int8 bucket; confidence
≡ 0 across all anchors.

**QAT addresses weight quantization noise, not output-tensor activation
quantization on a mixed-range head.** Even with fake-quant nodes simulating
int8 round-trips during the fine-tune, the *export-time* per-tensor
activation scale on the final output tensor is computed by the
TFLiteConverter from calibration data — exactly the same way as in PTQ.
The numeric outcome is therefore identical.

To actually recover, one of the following is required (all out of scope for
US-005):

1. **Detect-head split** (preferred): emit `cls` and `xywh` as *separate*
   output tensors so each gets its own per-tensor scale. This is what
   ultralytics does internally in `model.export(format='tflite', int8=True)`,
   but it requires modifying ultralytics' export graph.
2. **Knowledge distillation to a smaller student** (US-006): a smaller
   student head may quantize more cleanly; orthogonal to US-005.
3. **More than 1 epoch + per-channel activation quant**: per-channel
   activation quantization (where supported by the runtime) would let cls
   keep its own scale even on a shared output tensor. TFLM doesn't support
   this on the ESP32-S3; we'd be optimizing for a runtime that's not our
   deployment target.

## What this changes for the rest of the PoC

- **US-006 (KD)**: now the *only* remaining lever in the YOLO family.
  Distillation may yield a student whose head quantizes cleanly (smaller
  cls activation range, etc.) even without a head-split.
- **US-007–US-009 (NanoDet)**: NanoDet-Plus uses a Generalized Focal Loss
  head with a separate distribution-encoded localization output, which is
  numerically cleaner under PTQ. Expectation: NanoDet's quantization gap
  may be closer to typical (a few mAP points) rather than catastrophic.
- **US-011 (TFLM bench)**: still bench `yolov8n_cat_int8.tflite` and
  `yolov8n_cat_qat_int8.tflite` — same-architecture, same-size — to
  confirm that arena and op breakdown are unchanged. This is a useful
  control even when accuracy didn't move.
- **US-012 (SUMMARY decision matrix)**: the YOLOv8n-cat row will be
  reported with `mAP=0` honestly. The recommendation will lean toward
  whichever of {KD-shrunk YOLOv8n, NanoDet-cat} actually quantizes
  cleanly.

## Reproduce

```bash
# requires CUDA + ~3 minutes (1 epoch GPU + PTQ pipeline)
python -m training.edge.yolo.qat_finetune \
    --us003 training/edge/results/US-003.json \
    --us004 training/edge/results/US-004.json \
    --base  training/edge/models/yolov8n_cat.pt \
    --epochs 1 --lr 1e-4 --imgsz 224 --batch 32 --device 0 \
    --out training/edge/models/yolov8n_cat_qat_int8.tflite \
    --results-dir training/edge/results

# inspect the JSON
cat training/edge/results/US-005.json
```

The script is also safe to run in the SKIP path: pass a US-004.json with
`map50` close to US-003's and the orchestrator emits a `status='skipped'`
JSON without touching ultralytics or TF.
