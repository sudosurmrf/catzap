# US-003 — Fine-tune YOLOv8n single-class (cat-only)

Fine-tunes the stock COCO `yolov8n.pt` on the US-002 auto-labeled cat
dataset (480 train / 120 val) for 50 epochs at imgsz=224, batch=32,
`single_cls=True`, on CUDA. Output is copied to the canonical
`training/edge/models/yolov8n_cat.pt` path that downstream stories
(US-004 PTQ, US-006 KD, US-011 TFLM bench) consume.

## Result vs the stock checkpoint (apples-to-apples on the same val)

The PRD asks for a comparison vs `US-001-yolov8n-baseline-224`. That
baseline was scored on a 22-image silver GT bootstrapped from

labeled set used here. To show the fine-tune effect honestly, the table

## What changed structurally

- Detection head was reshaped to `nc=1` (single class "cat") via
  ultralytics' `single_cls=True` flag. This is what saves the ~146 K
  params and the ~5 % size drop — the multi-class classification
  outputs are simply not present.
- Conv backbone weights were initialized from the stock `yolov8n.pt`
  COCO checkpoint, then fine-tuned for 50 epochs on the auto-labeled
  set. Final epoch ultralytics-internal val mAP was 0.995 (much higher
  than our eval-harness 0.909 — discussed below).

## Why the eval-harness mAP (0.909) is lower than ultralytics' own (0.995)

Two reasons, both expected:

1. **Default confidence threshold**: our `YoloPytorchAdapter` applies a
   confidence floor of 0.25 before scoring (it's the production
   threshold mirrored from `server/vision/detector.py`). Ultralytics'
   internal validator computes mAP across the full PR curve and so is
   higher.
2. **Greedy single-class IoU matching**: our `compute_map50` is a
   simpler greedy-match VOC-style metric, also citing only the IoU=0.5
   point. Ultralytics' internal AP computation is more lenient.

Both methods rank fine-tuned > stock by a wide margin, which is the
result US-003 actually needs to demonstrate.

## Reproduce

```bash
# 1. Ensure US-002 labels + jpgs are present; if not:
python -m training.edge.auto_label bootstrap
python -m training.edge.make_dataset_manifest

# 2. Fine-tune (CUDA recommended; ~70 s on RTX 4090):
python -m training.edge.yolo.train_yolov8n_cat \
    --base yolov8n.pt --epochs 50 --imgsz 224 --batch 32 --device 0 \
    --output training/edge/models/yolov8n_cat.pt

# 3. Score with the harness:
python -m training.edge.eval.run_eval \
    --model training/edge/models/yolov8n_cat.pt \
    --format pytorch \
    --story-id US-003 \
    --val-dir training/edge/data/labeled/val \
    --imgsz 224
```
